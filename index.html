Gargle - Primary Contributor to Website and Application

Dental Marketing Platform

One of the primary offerings of Gargle is an up and coming web platform that allows practices to receive insight and analytics into the success of their dental practice's marketing efforts. I am the assigned owner of this project and built all the features on the frontend and have been a strong contributor to the microservice API's. The frontend has been built in Vue.js and the API's are written in either Go or Python (using Flask).

Some of the features I have built are:
- User authorization using Auth0
- A Direct Mail dashboard that includes building various charts using D3.js or Chart.js and involve a significant amount of user interaction and data manipulation
- A payment portal integration with a payment processor
- A Geographic analytics map that is integrated with Google maps using their heatmaps, map markers, and polygons
- A Resource Hub that enables the viewing and creating of articles via an in-house rich content editor that we built using Quill.js
- A user activity log as a piece towards HIPAA compliance
- A permissions system that is tied between our CRM, web application, and Auth0
- An in-house issue reporter that integrates directly with our engineering team's issue management application.
- I was a major contributor to an in-house, custom short URL generator


At Gargle, I also rebuilt their website which was originally built in Angular and is now built with Nuxt.js which has increased our SEO capabilities and improved performance.

I have also been one of the primary contributors to managing all things in GitLab. This includes:
- Setting up CI/CD pipelines that integrate with our deployments in GCP (Google Cloud Platform)
- Creating private npm packages using their npm package registry
- Setting up our engineering team's workflow processes using a slightly modified approach to Kanban and Scrum

I have also been a primary contributor to setting up our docker/kubernetes deployments on Google Cloud Platform
